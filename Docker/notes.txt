CONCEPT: Docker container

# WHAT IT DOES: Lightweight isolated environment for applications.
# Container = process, not a full VM.

# COMMANDs:

# Pull an image from Docker Hub
$ docker pull <image>

# Run a container in detached mode (-d), remove on exit (--rm), with custom name
$ docker run -d --rm --name busyboxtest busybox sleep 3600

# List running containers
$ docker ps

# Stop a running container
$ docker stop busyboxtest

# Run nginx container with port mapping (host:container)
$ docker run -d -p 8080:80 nginx


################## Building Images Using Dockerfile

# Check project3 for Dockerfile sample

# Build an image from a Dockerfile in the current directory
$ docker build --tag testbuild:v1 .
$ docker images
$ docker run -d --rm --name testbuild testbuild:v1 bash

# Best Practices & Tips:
# 1. Create a dedicated directory (e.g., /app/) containing only the files needed for your application.
#    When copying directory contents to the container's working directory, Docker will copy everything,
#    which unnecessarily increases the image size.
# 2. Create a .dockerignore file to exclude unnecessary files and directories from the build context.
# 3. Consider Dockerfile best practices (will work on this next).
